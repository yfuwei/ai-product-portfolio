Overview

一个重点项目，三个补充项目

本地 RAG 是主展示项目，完整体现问题定义、系统设计、GUI、验证与迁移。其他三个项目用于补充内容生成、企业知识检索和B端流程产品能力。

Project 01 · Featured

本地个人知识库 RAG 系统

面向个人笔记、项目资料和本地文档分散的问题，设计一个不上传私密资料的本地语义检索层，并提供可日常使用的 Web GUI。

Project 02

企业知识库问答与报告生成

面向资料分散、引用难追溯、初稿产出慢的问题，设计带引用溯源和人工复核机制的RAG问答与文本生成流程。

Project 03

AIGC视觉内容生成工作流

面向前期方案表达中出图周期长、需求口径抽象、反复修改成本高的问题，设计结构化输入、Prompt模板和多版本生成流程。

Project 04

B端审批协同平台工作流优化

面向报审材料反复、会前预审低效、评审意见难沉淀的问题，将线下沟通和人工追踪流程转为可提交、可预审、可追踪、可沉淀的协同流程。

About

苑夫唯｜从复杂项目协同转向 AI 应用产品

我拥有 UC Berkeley 建筑学硕士背景和旧金山 5 年学习与工作经历，过往工作集中在建筑设计、地产产品管理、方案评审和复杂流程协同。现在转向 AI 应用产品方向，重点关注 RAG、AIGC 和工作流自动化在真实业务场景中的落地。

复杂业务与流程经验

过往工作长期面对多角色协同、方案评审、设计变更、标准制度和项目推进等复杂场景，熟悉从模糊诉求到结构化流程、规则和交付结果的转化过程。

AI 应用产品实践

近阶段围绕本地个人知识库 RAG、企业知识库问答、AIGC 视觉内容生成和 B 端审批协同，独立完成需求拆解、流程设计、原型表达、验证问题集和效果评估。

我的产品优势

我更擅长把真实业务问题拆成可理解、可演示、可评估的产品方案：既关注用户路径和交互表达，也关注 AI 输出的边界、引用依据、人工复核和落地约束。
