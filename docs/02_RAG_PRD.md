# 企业知识库问答与报告生成 PRD 草稿

## 1. 项目背景

标准检索、案例复用、投前评审口径整理和汇报材料准备高度依赖分散资料和个人经验。直接使用通用大模型容易出现无依据生成和引用不可追溯问题。

## 2. 产品目标

设计带引用溯源和人工复核机制的知识库问答与文本初稿生成流程，把 AI 输出约束在可追溯资料范围内。

## 3. 目标用户

- 产品经理
- 投前评审人员
- 方案汇报材料准备人员
- 管理层决策支持人员

## 4. 核心流程

1. 选择资料库
2. 输入问题
3. 检索相关片段
4. 基于片段生成答案或初稿
5. 展示引用来源
6. 标记需人工复核内容
7. 生成可编辑文本初稿

## 5. Demo 范围

当前 Demo 使用预设资料和问题模拟 RAG 效果，不接入真实向量数据库。Demo 覆盖：

- 资料库范围展示
- 预设问题选择
- 召回片段展示
- 生成答案展示
- 引用来源展示
- 人工复核提示
- 汇报初稿生成

## 6. 验证口径

- 20 个验证问题
- 评估维度：检索相关性、答案可用性、引用准确性、人工修改成本
- 结果：19 个问题达到“基本可用及以上”

## 7. Demo 问题类型

| 类型 | 示例问题 | 产品目的 |
| --- | --- | --- |
| 事实检索 | 集团商办标准里，对公共空间的核心要求有哪些？ | 验证标准条文检索和要点总结 |
| 事实检索 | 区域产品管理办法中，对设计变更审批的限制条件是什么？ | 验证规则限制类问题回答 |
| 定向提取 | 请提取产品决策会材料里与方案可行性相关的关键条目 | 验证单文档提取能力 |
| 定向提取 | 区域产品汇报通常需要准备哪些输入资料？ | 验证结构化资料清单生成 |
| 对比归纳 | 集团标准和区域口径对同一问题的表述差异在哪里？ | 验证带依据的对比表达 |
| 对比归纳 | 不同TOD项目中有哪些共通做法和差异点？ | 验证跨案例归纳边界 |
| 文本生成 | 基于现有资料生成一段项目产品定位说明 | 验证资料驱动文本初稿 |
| 开放追问 | 对没有明确信息支持的评审问题怎么回答？ | 验证拒答和边界提示机制 |

## 8. 边界判断

适合：事实检索、单文档提取、标准条文查找、短文本初稿。

需谨慎：跨多文档复杂归纳、多案例异同比较、开放性结论判断。
