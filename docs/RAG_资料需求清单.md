# RAG Demo 资料需求清单

这份清单用于把“企业知识库问答与报告生成”Demo 从模拟样例升级为更贴近真实面试展示的版本。

## 1. 必须提供

### 1.1 脱敏资料片段 5-8 段

每段 200-500 字即可，不需要完整公司文件。

建议类型：

| 类型 | 需要数量 | 用途 |
| --- | --- | --- |
| 标准制度片段 | 1-2 段 | 做事实检索和引用来源 |
| 项目案例片段 | 1-2 段 | 做案例复用和产品定位说明 |
| 会议纪要/评审意见片段 | 1-2 段 | 做关键条目提取和评审问题归纳 |
| 汇报材料片段 | 1 段 | 做文本初稿生成 |
| 竞品分析片段 | 1 段 | 做对比归纳和摘要生成 |

### 1.2 你希望展示的 8-10 个问题

如果你不确定问题怎么写，可以直接给原始任务，我来改成面试可演示问题。

建议覆盖：

- 事实检索：查标准、查定义、查限制条件
- 定向提取：从会议纪要或材料中提取关键条目
- 对比归纳：比较两个口径、两个案例或两个版本
- 文本生成：生成产品定位、投前评审背景、汇报摘要
- 边界提示：资料不足、资料冲突、无明确依据时如何回答

### 1.3 可以公开展示的项目命名口径

你需要决定 Demo 中是否使用真实名称。

建议默认使用脱敏名称：

- `某集团产品标准`
- `区域产品管理办法`
- `TOD车辆段上盖项目指引`
- `高端住宅案例A`
- `产品评审会议纪要样例`

## 2. 可选提供

### 2.1 一张真实或模拟的资料截图

用于增强作品集可信度。可以是完全脱敏后的截图，只保留标题、章节、段落结构。

### 2.2 一张你希望模拟的页面参考图

比如你喜欢的知识库问答、Notion AI、飞书知识问答、企业搜索页面截图。

### 2.3 你希望面试官重点看到的能力

从下面选择 2-3 个即可：

- RAG 产品理解
- 引用溯源设计
- 人工复核机制
- 资料权限意识
- 文本初稿生成
- 企业知识库场景判断
- AI 边界判断

## 3. 脱敏规则

请删除或替换：

- 公司内部真实项目名称
- 具体地块名称
- 未公开财务数据
- 客户名称
- 供应商名称
- 内部人员姓名
- 具体合同、成本、售价、面积等敏感数字

可以保留或模拟：

- 资料类型
- 业务流程
- 章节结构
- 问题类型
- 判断逻辑
- 示例性结论

## 4. 推荐你直接发给我的格式

```text

标题：
正文：

资料2：项目案例片段
标题：
正文：

问题1：
希望答案重点：

问题2：
希望答案重点：
```

## 5. 我拿到资料后会做什么

1. 把资料整理成 Demo 的资料库卡片
2. 把问题改写成适合面试展示的预设问题
3. 为每个问题生成答案、召回片段、引用来源和人工复核提示
4. 更新 `src/data.js`
5. 更新 `docs/02_RAG_PRD.md`
6. 更新 `docs/RAG_验证问题集.md`
7. 如有截图或素材，放入 `public/images/rag/`
