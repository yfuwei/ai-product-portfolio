# 硬件约束

## 测试机器

- 2020 款 MacBook Pro。
- Intel i5。
- 16GB 内存。
- 无独立 GPU。

## 设计选择

- 使用小型本地 embedding 模型。
- 使用轻量本地 LLM。
- 使用 Chroma 本地向量库。
- 使用 FastAPI 作为轻量本地 Web 服务。
- 使用原生 HTML/CSS/JS，避免引入重型前端框架。

## 索引策略

- 优先索引 Obsidian Markdown。
- 本地原始文档按文件类型分批索引。
- 使用 `--offset` 和 `--limit` 分段运行。
- 使用 `--max-chunks` 控制单次任务规模。
- embedding 小批量提交，降低卡顿风险。

## 实际结果

在这台机器上，索引几百个 chunks 是可行的，但速度较慢。检索速度明显快于索引速度。

因此产品设计上把“分批索引”和“日常检索”区分开：索引可以慢慢跑，检索必须方便使用。
