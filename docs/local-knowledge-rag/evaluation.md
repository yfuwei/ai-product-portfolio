# 验证记录

## 验证目标

本项目优先验证检索质量，其次验证轻量回答质量。

系统被认为“有用”的标准是：

- 能在靠前结果中召回预期来源。
- 能展示可追溯来源路径。
- 能提供足够片段供人工复核。
- 只有在检索上下文支持时，才生成简短回答。

## 脱敏测试集

| 查询 | 预期来源 | 结果 |
|---|---|---|
| 长文档目录规则 | AI 协作规则笔记 | 命中 |
| 本地知识助手 GUI 启动方式 | RAG 日常使用手册 | 命中 |
| 文件管理规则 | 文件管理总览 | 命中 |
| 阅读标注导入流程 | 阅读标注导入指南 | 命中 |
| Agent 产品面试准备 | 面试准备笔记 | 命中 |
| 企业知识库 RAG 验证问题 | RAG 验证问题集 | 命中 |

## 观察到的优势

- 对明确规则、流程说明类笔记表现稳定。
- 当预期来源有清晰标题结构时，召回效果更好。
- 能帮助从大量 Obsidian 笔记中恢复项目上下文。
- Web GUI 降低了日常使用时对终端命令的依赖。

## 观察到的限制

- 在 Intel i5 机器上，本地 embedding 索引速度较慢。
- 轻量本地 LLM 对中英混合术语的理解不如云端大模型。
- 扫描 PDF 和 OCR 密集文件不在 MVP 范围内。
- 生成回答不能替代阅读来源片段。

## 示例检索结果

查询：

```text
文件管理规则
```

预期结果：

```text
sample-vault/notes/file-management.md
```

合理回答行为：

- 返回对应来源笔记。
- 展示关于“原始文件和笔记边界”的章节。
- 不编造检索上下文之外的制度。
