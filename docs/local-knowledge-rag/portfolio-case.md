# 作品集案例：本地个人知识库 RAG 系统

## 案例摘要

本地个人知识库 RAG 系统是一个面向个人笔记和本地文档的本地优先检索系统。它在 Obsidian 和本地文档库之上增加语义检索和带来源回答能力，同时不上传私人资料。

## 作品集定位

这个案例用于展示：

- AI 产品问题定义能力。
- 本地优先的隐私产品设计。
- RAG 工作流拆解。
- 索引和检索 pipeline 设计。
- 在低配置硬件约束下的产品取舍。
- 如何把私人项目封装成可公开展示的作品集材料。

## 用户场景

用户的知识分散在：

- Obsidian 笔记。
- 项目文档。
- 阅读笔记。
- 简历和面试材料。
- 本地 DOCX、PDF、PPTX、TXT、Markdown 文件。

用户需要通过语义方式找回历史上下文、规则和项目决策，而不是只能依赖关键词搜索。

## 产品设计

### 检索工作台

GUI 提供：

- 查询输入。
- 来源筛选。
- top-k 控制。
- 候选结果列表。
- 当前选中 chunk 详情。
- 来源路径和章节标题。

### 引用回答工作台

问答模式会先检索 chunks，再只基于这些 chunks 生成回答。回答被定位为“可复核初稿”，不是最终事实。

### 索引任务工作台

索引工作台暴露小批量控制项：

- 资料源。
- 文件类型。
- 文件数量限制。
- chunk 数量限制。
- 是否复用转换缓存。
- 是否重建 collection。

## 技术架构

| 层级 | 设计 |
|---|---|
| 来源层 | Obsidian Markdown 和选定本地文档 |
| 处理层 | 文件发现、文档转换、切块、本地 embedding |
| 存储层 | Chroma 向量库、可选转换缓存 |
| 交互层 | CLI 脚本和 FastAPI 原生 Web GUI |

## 隐私边界

公开作品集包含：

- 脱敏文档。
- 架构图。
- 配置示例。
- 验证摘要。
- 脱敏截图。

公开作品集不包含：

- 真实 Obsidian vault 内容。
- OneDrive 原始文档。
- Chroma 向量数据库。
- 转换后的文档缓存。
- 私人文件路径。
- 私人面试笔记。
- 私人准备材料。
