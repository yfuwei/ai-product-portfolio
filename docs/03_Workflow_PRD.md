# B端审批协同平台工作流优化 PRD 草稿

## 1. 项目背景

区域产品评审与方案报审流程中，材料提交、会前预审、评审意见反馈和后续追踪长期依赖邮件、微信群和共享文件夹，容易出现材料版本反复、预审效率低和评审意见难沉淀等问题。

## 2. 产品目标

将线下沟通和人工追踪流程转为平台化协同流程，实现可提交、可预审、可追踪、可沉淀。

## 3. 目标用户

- 提交方
- 预审方
- 评审方
- 管理者
- 平台管理员

## 4. 核心状态

1. 草稿
2. 已提交
3. 预审中
4. 退回补充
5. 待评审
6. 评审完成
7. 已归档

## 5. 核心模块

- 报审提报
- 材料版本管理
- 会前预审
- 评审意见记录
- 流程状态查看
- 角色权限
- 历史记录归档

## 6. Demo 范围

当前 Demo 展示一个固定报审案例的关键状态流转，不包含真实登录、真实上传和数据库。

## 7. 落地结果表达

围绕 6 个关键流程场景完成需求验收、联调测试与上线推进，将原本依赖线下沟通和人工追踪的报审流程转为可提交、可预审、可追踪、可沉淀的协同流程。
